# ドメイン共通要求仕様書

## 本書について

### 概要

<!-- HINT:
本書の位置づけ宣言。PRD(方針)・本書(横断要求)・ドメイン要求仕様書群(個別要求)との上下流関係を明示する。
ドメイン要求仕様書群(個別)はここで定義する `DOM-COMMON-*` を「継承する共通要求」として参照する関係になる。
-->

本書は、{{プロダクト名・サービス名}}のドメインに横断的に効く具体的な要求事項を集約したドキュメントです。

上流の[プロダクト要求仕様書(PRD)](./product-requirement-document.md)が「方針・概念・体験設計」を扱うのに対し、本書は **「3 ドメイン以上で横断的に効く具体的な要求」** を扱います。本書は[ドメイン定義書](./domain-definition-document.md)で定義された各ドメインに対するドメイン要求仕様書群(個別)の **親文書** として機能し、各ドメイン要求仕様書(個別)は本書の共通要求(`DOM-COMMON-*`)を「継承する共通要求」として参照する関係になります。

### 注記

本書では原則として How(具体的な実装手段)には踏み込みませんが、 **ビジネス・規制上譲れない具体水準**(可用性目標値・監査ログ保持期間・準拠すべき業界基準 等)は本書で確定します。これらは「実装手段の選択」ではなく「実装が満たすべき水準・基準」であり、本書のスコープに含まれます。
