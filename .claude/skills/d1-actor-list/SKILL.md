# D1: アクター一覧

## inputする成果物

| 成果物のパス | 内容 |
|---|---|
| `templates/D1.business-requirements/actor-list-template.md` | アクター一覧テンプレート(章構成・分類軸・書き方ヒント) |
| `docs/D1.business-requirements/business-requirement-document.md` | 上流のBRD(スコープ・対象ユーザー概要・コンプライアンス制約 等を参照) |
| `docs/D1.business-requirements/actor-list.md` | 既存アクター一覧(更新・レビュー時のみ参照。新規作成時は未存在) |

## outputする成果物

| 成果物のパス | 内容 |
|---|---|
| `docs/D1.business-requirements/actor-list.md` | アクター一覧本体 |

## 新規作成時の振る舞い

新規作成時(`docs/D1.business-requirements/actor-list.md` 未存在)は **ユーザーへのヒアリングを行わず**、上流の BRD と業界一般知識から人間アクターを推定して直接生成する。

1. BRD(`docs/D1.business-requirements/business-requirement-document.md`)を読み込み、スコープ・対象ユーザー概要・コンプライアンス制約 等から人間アクター候補を分類軸(種別・境界・関与の仕方)ごとに抽出する
2. 直接利用者・間接受け手・統制監督・関係部門 の各観点で網羅性を確認する
3. テンプレート(`templates/D1.business-requirements/actor-list-template.md`)の形式に従って書き出す。BRD・業界知識から確定できない値には要確認マーカーを付ける

## 留意事項

- **人間アクターのみを扱う**: システム等の非人間アクターは本書では扱わず、後続の C4 モデル(レベル1・2)で表現する。本書はシステムを直接利用する者だけでなく、システム出力物を他者経由で受け取る者・統制する者など、直接利用しない人間アクターも含めて網羅する
- **分類軸は固定**: 「種別・境界・関与の仕方」の3軸を基本とする。プロジェクト固有の分類軸を追加してもよいが、可読性のため4軸以内に留める
- **業務観点に留まる**: 機能名・画面名・テーブル名等のシステム実装語は混入させない。詳細仕様(具体的な権限・操作手順・利用シナリオ 等)は下流の PRD・ドメイン要求仕様書等に降ろす
- **BRD との整合性**: 上流の BRD のスコープ(対象業務範囲)・対象ユーザー概要と整合させる

## 完成時のセルフチェック

執筆完了時に以下の観点で出力を検査し、不備があれば修正したうえで出力を完了とする。

- 人間アクターのみが記載されているか(システムが紛れ込んでいないか)
- システムを直接使う人だけでなく、システム出力物を他者経由で受け取る人や、システムによって影響を受ける人も挙げられているか
- 重複なく、抜けなく(MECE)挙げられているか
- BRD のスコープ(対象業務範囲)・対象ユーザー概要と整合しているか
- 機能名・画面名・テーブル名等のシステム実装語が混入していないか(業務観点に留まっているか)
- `{{` や XXX のプレースホルダが残っていないか

## 下流への伝搬関係

作成・更新時は以下の下流成果物との整合を意識する:

| 下流成果物 | アクター一覧から何を引く |
|---|---|
| プロダクト要求仕様書 | 全般(特にシステムの利用ユーザー) |
| ユーザーストーリー一覧 | 全般(ユーザーストーリーのユーザーそのものがアクター一覧そのもの) |
| C4モデルレベル1 | システムの利用ユーザー |
| C4モデルレベル2 | システムの利用ユーザー |
| L1シーケンス | システムの利用ユーザー |
| L2シーケンス | システムの利用ユーザー |
