# 画面コンセプト集

## 画面体験のコンセプト

本プロダクトの画面体験は、紙・対面前提の現行新契約プロセスを **「関係者全員にとっての業務上のストレスを下げる電子化」** へと反転させることを核とします。営業職員・代理店募集人・引受査定担当者・新契約事務担当者・申込人・被保険者 のいずれにとっても、迷わず・つまずかず・証跡が自然に残る画面体験を目指します。紙起因の不備を入力時点で消し、引受判定の根拠を画面から追跡でき、対面でも非対面でも同じ動線で前進でき、お客様にとっては手続きの負担と不安が最小化される — このコンセプトをプロダクト全体の画面体験の通底とします。

## 体験原則

| ID | 原則名 | 内容 | 上流参照 |
|---|---|---|---|
| SC-PR-1 | 入力時点での不備抑止 | エラーの事後検知ではなく、入力時点でのチェック(必須項目・整合性・項目間矛盾)と前工程からの引き継ぎ精度の双方で不備の発生源を絶つ。後工程で「差し戻し → 修正 → 再提出」の往復を減らすことを画面体験の判断軸とする。 | PRD §「提供価値・体験方針(不備の発生抑止)」/ BRD KPI「不備返戻率5%以下」 |
| SC-PR-2 | 判定根拠の追跡可能性 | 引受査定の判定結果は結論と根拠が画面上で一体となり、後から第三者(コンプライアンス部・内部監査部・監督官庁)が遡及できる。自動判定と人手判定の差分・覆す理由も画面で明示される。判定の透明性を画面体験の中心に据える。 | PRD §「提供価値・体験方針(引受査定の標準化と透明性)」/ BRD KPI「自動判定率70%以上」 |
| SC-PR-3 | 対面/非対面の動線共通 | 募集人が対面(店舗・訪問先)で使う動線と、非対面(オンライン面談)で使う動線を **同じ画面・同じフロー** で完結させる。タブレットでの片手操作・ネットワーク不安定環境(訪問先)・狭い場所での操作を体験設計の前提に含める。 | PRD §「UI/UX方針(利用シーン)」/ PRD §「システム利用ユーザーと体験設計(ACT-1・ACT-2)」 |
| SC-PR-4 | 顧客接点のアクセシビリティと負担最小化 | 申込人・被保険者向け画面は WCAG 2.1 レベル AA を必須水準とし、端末リテラシーのばらつきが大きい一般消費者でも、本人確認・告知入力の負担が最小化される設計を目指す。要配慮個人情報の取得目的明示と同意取得は業務動線に自然に組み込む。 | PRD §「UI/UX方針(アクセシビリティ)」/ PRD §「システム利用ユーザーと体験設計(ACT-5・ACT-6)」 |
| SC-PR-5 | 募集コンプライアンスの体験埋め込み | 意向把握・情報提供・適合性原則・告知 等の募集コンプライアンス上の確認事象が、業務動線の中で **証跡として自然に残る** 画面設計とする。証跡化のために業務動線が遠回りになる作りは避け、「お客様本位の業務運営」と業務生産性の両立を画面体験の判断軸とする。 | PRD §「提供価値・体験方針(募集コンプライアンスの証跡化)」/ PRD §「法務・コンプライアンス要求(PRD-REG-1, PRD-REG-6)」 |

## 代表シーン

### SC-SCENE-1: 意向ヒアリングシート — 設計起点としての意向把握

* **場面**: 営業職員(ACT-1)・代理店募集人(ACT-2)が、対面(店舗・訪問先のタブレット)または非対面(オンライン面談)の中で、申込人(ACT-5)に対して意向ヒアリングを行う場面。新契約フローの最初の業務工程であり、後続(設計書作成・申込受付)へ引き渡される意向情報の起点。
* **このシーンで体現したいコンセプト**:
  - **SC-PR-5(募集コンプライアンスの体験埋め込み)** の入り口 — 意向把握が「形式的に項目を埋める作業」ではなく「お客様との対話で意向を引き出す業務体験」になっていること。
  - **SC-PR-1(入力時点での不備抑止)** — 紙のヒアリングシート起因の不備(項目漏れ・転記ミス)を入力時点で根絶する。
  - **SC-PR-3(対面/非対面の動線共通)** — 対面の場でも非対面のオンライン面談でも、同じ画面・同じ動線で完結する。
* **期待する画面の雰囲気・要素**:
  - タブレットを募集人とお客様の間に置き、両者で画面を覗き込める明快な情報構造。
  - 意向項目(目的・保障の種類/額/期間・予算・既契約・家族・職業 等)が業務会話の流れに沿った順序で配置。
  - 「暫定 → 確定」の状態が画面上で明示され、未確定のまま設計書作成に進めない仕掛けが自然に組み込まれている。
  - 個人情報取得の同意プロセスが業務動線に違和感なく挟まる。

### SC-SCENE-2: 設計書(重要事項)提示・確定 — 加入意思の核心

* **場面**: 営業職員(ACT-1)・代理店募集人(ACT-2)が、申込人(ACT-5)に対して、ヒアリング結果に基づいて作成した設計プランと重要事項を提示し、確定の合意を得る場面。意向把握 → 申込受付 をつなぐ業務動線の中核であり、お客様が「このプランで加入する」と意思を固める瞬間。
* **このシーンで体現したいコンセプト**:
  - **SC-PR-5(募集コンプライアンスの体験埋め込み)** を最も色濃く体現するシーン — 重要事項説明・適合性原則の確認・お客様の同意取得が業務動線に自然に組み込まれ、すべて証跡として残る。
  - **SC-PR-1(入力時点での不備抑止)** — 意向との整合性検証(乖離可視化)で、後工程の申込・告知段階に問題を持ち越さない。
  - **SC-PR-3(対面/非対面の動線共通)** — タブレットを介した対面の対話でも、オンライン面談での画面共有でも、同じ体験を提供。
* **期待する画面の雰囲気・要素**:
  - タブレットを募集人とお客様で覗き込む利用シーンを前提とした、視認性・指タップ操作性の高い情報構造。
  - 確定プラン(主契約・特約・期間・保険料)が一目で分かるサマリ + 詳細展開の二層構造。複数プランを比較しながら絞り込む動線も用意する。
  - 意向との整合性(意向把握で取得した目的・予算・希望保障 等との対比)が可視化され、乖離があれば明示。
  - 重要事項の説明事項(保険業法の情報提供義務に基づく事項)を、お客様と一緒に読み上げ・確認できるインライン提示。チェック行為が証跡として自然に残る。
  - 「お客様の確定意思を引き出す瞬間」としての儀式感 — 確定ボタンに重みがあり、確定後の遷移(申込受付への引き渡し)が明快。

### SC-SCENE-3: 申込情報入力 — 募集人とお客様の共同編集

* **場面**: 営業職員(ACT-1)・代理店募集人(ACT-2)と 申込人(ACT-5) が、確定した設計プランを起点として、申込情報(申込人・被保険者・受取人の関係性、払込方法、受取人指定 等)を共同で入力する場面。新契約フローの中でお客様が画面の前で最も長い時間を過ごす、実体験の中心となる業務工程。
* **このシーンで体現したいコンセプト**:
  - **SC-PR-1(入力時点での不備抑止)** を最も色濃く体現するシーン — 必須項目・項目間整合・関係性の構造化チェックが入力時点で働き、後工程の差し戻し(KPI 不備返戻率5%以下)を起こさない。
  - **SC-PR-4(顧客接点のアクセシビリティと負担最小化)** — お客様にとって「自分の情報が正しく入力されている」ことが画面から見える安心感を提供する。
  - **SC-PR-5(募集コンプライアンスの体験埋め込み)** — 被保険者同意の取得・本人確認(KYC)・反社チェック・電子署名 等の前提条件が業務動線の中で自然に充足される。
* **期待する画面の雰囲気・要素**:
  - タブレットを介した募集人とお客様の共同編集を前提とした、双方が読めて触れる情報配置。狭い場所での操作や訪問先のネットワーク不安定環境にも耐える。
  - 申込人・被保険者・受取人の三者の関係性が直感的に把握できる構造化入力(続柄・指定割合の関係図示)。被保険者同意要否の判定が画面上で行われる。
  - 前提条件(KYC・反社・告知・電子署名・募集権限)の充足状況が常に可視化され、「何が残っているか・次に何をすべきか」が明示される。
  - お客様自身が入力・確認した内容と、募集人が代理入力した内容が区別され、責任所在が透明。
  - 申込意思の確定は電子署名による儀式として位置づけ、確定までの導線が明快。
